## Decision

A9 MoE review is a lightweight requirements review council, not a numeric
score wheel.

Required viewpoints:

- why / real problem
- scope and dependencies
- system requirement translation
- tradeoff and architecture
- role boundary
- test verifiability
- requirement quality
- exception governance
- security / non-functional requirements
- execution governance
- product mainline pressure
- external learning
- data model
- performance depth

## Rule

The council is most important before execution. After decision closure, worker
execution should be narrow and evidence-driven.
